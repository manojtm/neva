Technologies Used:

  1. PHP CodeIgniter
  2. CodeIgniter REST Server
  3. Bootstrap
  4. jQuery for AJAX calls
  5. MySQL

Set up

Copy the directory to your webroot. Import SQL file. DB config file path: /api/application/config/database.php

Flow

As per the document, I have inserted 2 users. But you can register with new user to test from beginning.

  1. Registration: If email id does not exists, account will be created. Then we need to log in to proceed. Form shows relavent server responses. Like user exists, account creations successfull.
  2. Login: On correct auth credentials, page will be redirected to dashboard. Form will show relavent server response. Like login success, wrong username/password.
  3. On successfull login, I am saving user id in localStorage for session management. So, on logout, we must not access dashboard.html
  4. In Dashboard: Adding Friend: An account must exists to send a message. Relavent server response is privided. Ex: User does not exists, Friend is allready added you can send message, Friend addedd successfuly.
  5. In Dashboard: On page load list of friends are shows. After adding a new friend the same method is re-called to reload the list.
  6. In Dashboard: Clicking on Chat button will display the chats w.r.t the selected friend. We can send a message from the input box given. Clicking on Chat button again will refresh the chats. Auto refreshing of chats is not yet done. The chat will be re-loaded once we send the message.
  7. Logout: Will clear the session in server, then clears localStorage
  8. Password are stored as MD5.
  9. Relavent MySQL fields are indexed.
  10. RESTful architecture.
